# v0 Prompt — Calex Frontend

Build the complete frontend for **"Calex"** — an AI DevRel Employee-as-a-Service. This is a YC hackathon project. The goal: in under 2 minutes, make judges believe "This company just hired an AI DevRel employee and it's already working."

## Stack

- Next.js 14+ (App Router) + TypeScript + Tailwind CSS
- Dark-mode first. Background: `#0a0a0f`, cards: `#12121a`, accent: `#7c3aed` / `#a78bfa`
- Use `lucide-react` for icons
- shadcn/ui components where appropriate
- Minimal, clean, premium SaaS aesthetic
- All API calls go to `process.env.NEXT_PUBLIC_API_URL` (default `http://localhost:3001`)

## What Calex Does (context for UI copy)

Calex is a DevRel "employee" that a company hires. It:
- Answers developer questions via a website chat widget (like Intercom)
- Can do live "Show me" walkthroughs — an AI browser agent navigates the company's docs/site and returns step-by-step instructions with voice narration
- Responds to Discord/Slack community questions automatically
- Has its own email inbox (AgentMail) and phone/SMS number (Twilio)
- Learns from every interaction and builds playbooks over time
- Shows all activity in a founder dashboard

Sponsor tech used (label accurately in UI):
- **Browser Use** = guided walkthrough automation + run logs
- **Supermemory** = persistent agent memory & context
- **AgentMail** = agent email inbox identity
- **Vapi** = AI voice agent with phone number (developers can call it)

### 2. Onboarding Wizard — `/app/onboarding`

Multi-step form to "hire" your DevRel agent.

**Step 1 — Company Info:**
- `company_name` (text, required)
- `website_url` (url, required)
- `docs_urls` (textarea, optional — comma-separated URLs)
- `founder_email` (email, required)

**Step 2 — Agent Identity:**
- `agent_name` (text, required — e.g. "Aria", "Dev", "Max")
- `tone` (select: "friendly", "professional", "casual", "technical")
- `demo_mode` toggle (default: ON) with helper text: "Uses deterministic responses for reliable demos"

**On submit → call API:**
```
POST /api/onboard
Body: {
  company_name: string,
  website_url: string,
  docs_urls: string[],       // optional
  agent_name: string,
  tone: string,
  founder_email: string,
  demo_mode: boolean
}
Response: {
  company_id: string,
  slug: string,
  agentmail_address: string,
  vapi_phone_number: string,
  demo_mode: boolean
}
```

**While waiting — show animated progress screen:**
- "Crawling website..." (with spinner)
- "Building knowledge memory..." 
- "Creating email identity..."
- "Setting up phone number..."
- "Your DevRel agent is ready!"

These are purely animated UI states (show them sequentially with ~2s delays). The actual API call is a single POST that returns when done.

**Success screen shows:**
- Agent name + company name
- AgentMail address (copyable)
- Vapi phone number (copyable) with hint: "Developers can call this number to talk to your agent"
- Two buttons: "Open Dashboard" → `/app` and "Open Help Page" → `/help/[slug]`

**Store `company_id` and `slug` in localStorage** for use across pages.

### 3. Dashboard — `/app`

This is the founder's control center. It should feel like a live operations dashboard.

**Top bar:** "Calex Dashboard" + company name + agent name (from localStorage or fetched)

**Metric cards row (4 cards):**
- Questions Answered (today count)
- Walkthroughs Run (today count)
- Escalations (today count)  
- Avg Response Time

These can be derived client-side from the ActivityFeed data. Count activities by type for today.

**Main section: Live Activity Feed**
```
GET /api/activity?company_id={company_id}
Response: ActivityFeedItem[]

interface ActivityFeedItem {
  company_id: string
  ts: string              // ISO timestamp
  type: string            // "widget_answered" | "walkthrough_executed" | "community_answered" | "escalated" | "onboarding_started" | "memory_built" | "identity_created" | "setup_complete"
  title: string           // e.g. "Answered widget question"
  detail: string          // e.g. "How do I install the SDK?"
  status: string          // "success" | "pending" | "error"
  artifact_urls?: string[]
  session_id?: string     // present for walkthrough events
  meta_json?: any
}
```

**Poll this endpoint every 3 seconds.** Newest items at top. Animate new items sliding in.

Each activity item renders:
- Icon based on `type` (MessageSquare for widget, Globe for walkthrough, Hash for community, AlertTriangle for escalation, etc.)
- `title` bolded
- `detail` as secondary text
- Relative timestamp ("2m ago")
- Status badge (green for success, yellow for pending, red for error)
- If `session_id` exists: show a **"Watch"** button that opens the WalkthroughModal

**Demo trigger buttons** (bottom of page or in a collapsible sidebar):
- "Simulate Discord Question" — opens a small form: message text input + submit
  ```
  POST /api/demo/community_event
  Body: {
    company_id: string,
    platform: "discord",
    channel: "general",
    user_id: "demo-user",
    message_text: string
  }
  Response: { status: "ok" }
  ```
- "Simulate Slack Question" — same but platform: "slack"

After triggering, the activity feed should pick up the new events on next poll.

### 4. Hosted Help Page — `/help/[companySlug]`

This simulates "Calex embedded on a company's website." It's what you'd show to prove the widget works.

**Layout:**
- Left side or top: Company header bar showing company name
- Main area: the CalexWidgetChat component (not a tiny floating bubble — make it prominent, center-stage)

**CalexWidgetChat component:**
- Chat-style interface
- Input field at bottom: "Ask a question about [company_name]..."
- Two action buttons next to input: **"Answer"** and **"Show Me"**
- Messages render in a scrollable chat area

**"Answer" mode:**
```
POST /api/widget/ask
Body: {
  company_id: string,
  question: string,
  requested_mode: "answer"
}
Response: {
  mode: "answer",
  answer_text: string,
  confidence: number,        // 0-1
  suggested_links: string[]
}
```
Render `answer_text` as a chat bubble from Calex. Show `suggested_links` as clickable chips below the answer.

**"Show Me" mode:**
```
POST /api/widget/ask
Body: {
  company_id: string,
  question: string,
  requested_mode: "show"
}
Response: {
  mode: "show",
  answer_text: string,
  steps: string[],           // ordered walkthrough steps
  final_url: string,
  session_id: string,
  narration_script: string,
  narration_audio_url: string | null  // MiniMax TTS mp3 URL (valid 24h)
}
```
Render:
1. `answer_text` as a chat bubble
2. `steps` as a numbered list with checkmark styling
3. `final_url` as a clickable link
4. **Automatically open the WalkthroughModal** with the returned `session_id`

Show a loading state ("Calex is navigating...") while the API call is in flight.

### 5. WalkthroughModal (shared component)

A modal/drawer that shows a Browser Use walkthrough session.

**Input:** `session_id` (string) and optionally `narration_script` (string)

**Fetch:**
```
GET /api/browser/run?session_id={session_id}
Response: {
  company_id: string,
  session_id: string,
  purpose: string,
  status: string,           // "running" | "completed" | "failed"
  run_log: string[],        // timestamped strings like "[00:02] Navigated to docs page"
  preview_frames: string[], // optional, URLs to screenshots
  final_url: string,
  created_at: string
}
```

**Poll every 2 seconds** while `status === "running"`.

**Render:**
- Modal header: "Live Walkthrough" + purpose
- Status badge (running/completed/failed)
- `run_log` as a live-updating terminal-style list. New entries animate in. Use a monospace or code-style font. Auto-scroll to bottom.
- If `preview_frames` exist, show the latest as a small image preview
- `final_url` as a clickable link when completed
- **"Play Narration"** button: if `narration_audio_url` exists, play it as an `<audio>` element (MiniMax TTS mp3). Fallback to browser `SpeechSynthesis` if URL is null.
- Close button

## SHARED COMPONENTS

### ActivityFeed
- Props: `companyId: string`
- Polls `GET /api/activity?company_id=` every 3s
- Renders list of ActivityItem components
- Shows empty state if no items

### ActivityItem
- Props: `ActivityFeedItem`
- Icon + title + detail + timestamp + status badge
- Optional "Watch" button if `session_id` present

### MetricCard
- Props: `title: string, value: string | number, icon: LucideIcon, trend?: string`
- Simple card with icon, big number, label

### CalexWidgetChat
- Props: `companyId: string, companyName?: string`
- Full chat interface with answer/show-me modes
- Manages message history in local state

### WalkthroughModal
- Props: `sessionId: string, narrationScript?: string, onClose: () => void`
- Fetches + polls browser run data
- Renders run_log + narration button

### OnboardingWizard
- Multi-step form component
- Manages step state internally
- Calls onboard API and shows progress

## DESIGN GUIDELINES

- **Dark mode only.** Background `#0a0a0f`, surface `#12121a`, borders `#1e1e2e`
- **Accent purple:** `#7c3aed` (buttons, highlights), `#a78bfa` (lighter accents)
- **Typography:** Inter or system-ui. Headings bold, body regular.
- **Cards:** Rounded corners (12px), subtle border, no heavy shadows
- **Animations:** Subtle fade-ins for new activity items. Smooth transitions between onboarding steps. Pulse animation for "working" states.
- **Status colors:** Green `#10b981` (success), Yellow `#f59e0b` (pending), Red `#ef4444` (error)
- Badge styles: pill-shaped, translucent background with colored text
- **Premium SaaS feel.** Think Linear, Vercel dashboard, Raycast.
- **Responsive** but optimize for desktop-first (hackathon demo will be on a laptop)
